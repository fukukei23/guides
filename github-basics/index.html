GitHub 基礎ガイド

GitHubとは

コードをインターネット上に保存・共有・管理するサービス。
「Gitの履歴」をクラウドに置いて、チームで共有したり、自動テストを走らせたりできる。

Git = バージョン管理ツール（ローカル）／GitHub = それを置くクラウドサービス（リモート）

基本用語

用語意味（素人向け）
リポジトリ（repo）プロジェクトの保管庫。コードの履歴ごと入っている
コミット「この状態を保存した」という記録。セーブポイント
ブランチ本流を壊さずに変更を試せる作業コピー
プルリクエスト（PR）「このブランチの変更を本流に取り込んでいいですか？」という申請
マージブランチの変更を本流に取り込む操作
Issueバグ報告・タスク・相談を管理するチケット
Fork他人のリポジトリを自分のアカウントにコピー
CloneGitHubのリポジトリを自分のPCにダウンロード
Actionsコードをpushしたら自動でテスト・デプロイなどを走らせる仕組み
webhook特定のイベントが起きたら指定したURLに自動通知する仕組み

コードが世に出るまでの流れ

ブランチ作成

コード変更

push

よく見る画面

画面何をする場所か
Code タブファイル一覧・最新のコードを確認
Issues タブタスク・バグのチケット管理
Pull requests タブ変更申請の一覧・レビュー
Actions タブ自動テスト・デプロイのログ確認
Settings タブリポジトリの設定（権限・webhook等）

gh コマンド（GitHub CLI）とは

上記の操作をターミナルから文字で実行できる公式ツール。
ブラウザを開かずに Issue作成・PR確認・CI結果確認ができる。

やりたいことgh コマンド
Issue作成gh issue create
PR作成gh pr create
CI結果を見るgh run watch
PRをマージgh pr merge --squash
→ コマンド一覧は GitHub CLI 逆引き を参照

MCPサーバー vs gh コマンド

ClaudeからGitHubを操作する方法は2つある。

方式仕組みコンテキスト消費使いどころ
GitHub MCPMCPサーバーが常時起動してClaudeと橋渡し多い（常時接続）「このIssue見て」と会話しながら操作
gh コマンドスクリプトがGitHub APIを直接叩く少ないIssue作成・PR確認などの定型処理
定型操作はスクリプトに gh コマンドを書いておく → MCP不要 → コンテキスト節約

webhook（Discordへの通知など）

「特定のイベントが起きたら自動で通知する」仕組み。Discord への完了通知などに使う。

Discord に通知を送る（curlで直接）
curl -X POST "$DISCORD_WEBHOOK_URL" -H "Content-Type: application/json" -d '{"content":"作業完了しました"}'
Discord MCP（MCPサーバー）を使わなくてもこれだけで通知できる → MCP不要
⚠️ webhook URLは秘密情報。.secrets.env に保存してコードに直接書かない
